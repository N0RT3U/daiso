## 2. 대시보드 1: 다이소 런칭 시뮬레이터

### 2.1 목적

신규 브랜드/제품이 다이소에 입점할 경우의 **연착륙 가능성**을 8축 점수 체계로 시뮬레이션한다. 사용자가 카테고리, 가격, 성분 수, 브랜드 인지도 등의 파라미터를 입력하면 실시간으로 총점과 등급(A/B/C/D)을 산출한다.

## 3. 8축 점수 체계

### 3.1 구조 개요

총점 100점 만점으로, 8개 축의 가중합으로 산출한다.

| 축 | 항목 | 만점 | 산출 방식 |
|----|------|------|----------|
| 축1 | 카테고리 적합성 | 30 | `category_score_max30` (BigQuery 뷰에서 사전 계산) |
| 축2 | 브랜드 인지도 | 25 | 4개 질문 점수 합산 → `ROUND((Q1+Q2+Q3+Q4)/12*9+1)` → 1~10 → `(점수-1)/9*25` |
| 축3 | 가격 적합성 | 15 | 입력 가격과 SL 제품 중앙값 비교 (편차율 기반 5단계) |
| 축4 | 성분 수 적합성 | 15 | 입력 성분 수와 SL 제품 중앙값 비율 비교 (0.6 이하 → 15점) |
| 축5 | 성분 적합성 (골든) | 20 | `MIN(Gold매칭수*2 + Silver매칭수*1, 20)` |
| 축6 | 기능성 여부 | 5 | 기능성이면 5, 아니면 0 |
| 축7 | 성분 타입 적합성 | 15 | SLI DNA 기반 (식물추출물 15 > 왁스/폴리올 12 > 비타민/산 9 > 혼합 5 > 합성폴리머 2) |
| 축8 | 안티 성분 감점 | -20 (감점) | `MAX(-Red수*2 - Orange수*1, -20)` |

### 3.2 등급 판정

총점을 카테고리 내 백분위로 환산한 후, 다음 기준으로 등급을 부여한다.

| 등급 | 백분위 조건 | 의미 |
|------|-----------|------|
| A | 상위 10% 이내 | 입점 적극 추천, 연착륙 가능성 높음 |
| B | 상위 10~30% | 입점 추천, 일부 개선 시 경쟁력 확보 |
| C | 상위 30~60% | 조건부 입점, 개선 필요 |
| D | 하위 40% | 입점 재검토, 전략 수정 필요 |

### 3.3 핵심 계산식

**총점:**
```
축1 + 축2 + 축3 + 축4 + 축5 + 축6 + 축7 + 축8(감점)
```

**축2 브랜드 인지도 (자동 산출):**
```
ROUND((유통경험점수 + 바이럴수준점수 + 검색노출점수 + 리뷰축적도점수) / 12 * 9 + 1)
→ 1~10점 스케일
→ (점수 - 1) / 9 * 25 → 0~25점
```

**축3 가격 적합성:**
```
편차율 = ABS(입력가격 - SL중앙값) / SL중앙값
편차율 0% → 15점 / ≤20% → 12점 / ≤40% → 9점 / ≤60% → 6점 / >60% → 3점
```

**축8 안티 성분 감점:**
```
MAX(-Red수 * 2 - Orange수 * 1, -20)
달성률 = (20 + 감점) / 20
```

## 4. 대시보드 2: 다이소 뷰티 Safety Monitor

### 4.1 목적

기존 입점 브랜드/제품의 **감성 위험**을 실시간 모니터링한다. ABSA(Aspect-Based Sentiment Analysis) 결과와 성분 위험도(Z-Score)를 교차 분석하여 경고/주의 제품을 조기에 식별한다.

### 4.2 레이아웃

```
┌──────────────────────────────────────────────────────────┐
│                    헤더 (50px, #cf4948)                    │
├──────────────┬───────────────┬──────────────┬────────────┤
│ KPI 전체부정률│ KPI 감성악화  │ KPI 연착륙비율│ KPI 긍정률  │
├──────────────┴───────────────┼──────────────┴────────────┤
│                              │                           │
│   Z-Score 구간별 제품 수      │   성분 위험 vs 감성 위험   │
│   (히스토그램)                │   (산점도)                 │
│                              │                           │
├──────────────────────────────┼───────────────────────────┤
│                              │                           │
│   월별 부정률 추이            │  Aspect x Category        │
│   (라인 차트)                │  부정률 히트맵              │
│                              │                           │
├──────────────────────────────┼───────────────────────────┤
│                              │                           │
│   브랜드별 부정률             │  상위 제품 Engagement 순위 │

│                              │                           │

```

### 4.4 매개변수 (Safety)

| 매개변수 | 타입 | 설명 |
|---------|------|------|
| p_zscore_threshold | real (range) | Z-Score 경고 임계값 |
| p_sentiment_threshold | real (range) | 부정률 감성악화 임계값 |
| p_trend_period | string (list) | 추이 분석 기간 |
| p_alert_lookback | integer (range) | 경고 산출 참조 기간 |

### 4.5 Safety 핵심 계산식 (SF_ 시리즈)

| 계산식 | 로직 |
|--------|------|
| SF_부정비율_제품 | `SUM(neg_count) / SUM(mention_count)` |
| SF_부정_ZScore | `(AVG(neg_pct) - AVG(cat_avg_neg_pct)) / WINDOW_STDEV(AVG(neg_pct))` |
| SF_부정경고 | Z-Score >= 임계값 → "경고", >= 임계값x0.67 → "주의", else "정상" |
| SF_감성악화 | `AVG(neg_pct)/100 > p_sentiment_threshold` → "악화" |
| SF_위험등급 | 경고+악화 → High, 경고 or 악화 → Medium, 주의 → Low, else Normal |
| SF_Aspect부정_색상 | Aspect별 차등 임계값 적용 (재질/냄새 25%, 배송/포장 23%, 용량/휴대 20% 등) |
| SF_재구매언급률 | `FIXED [product_code]: AVG(IF aspect='재구매' THEN 언급률 END)` |
| SF_가성비만족도 | 가격/가성비 Aspect 긍정률 |
| SF_카테고리부정비중 | 카테고리별 부정 건수의 전체 비중 |

## 5. 대시보드 3: 브랜드 자가진단 대시보드

### 5.1 목적

특정 브랜드/제품을 선택하여 **SWOT 분석 결과**와 **카테고리 내 경쟁 포지션**을 시각적으로 확인한다. ABSA 감성 분석 기반으로 Strength/Weakness/Opportunity/Threat를 자동 산출한다.

### 5.2 레이아웃

```
┌──────────────────────────────────────────────────────────┐
│                    헤더 (50px, #cf4948)                    │
├──────────────────────────────────────────────────────────┤
│                   [제품 선택기]                            │
├──────────────┬──────────────┬──────────────┬─────────────┤
│   강점 (S)   │  약점 (W)    │  기회 (O)    │  위협 (T)   │
├──────────────┴──────────────┴──────────────┴─────────────┤
│                                                          │
│         내 제품 vs 카테고리 평균 (비교 차트)               │
│                                                          │
├────────────────────────────┬─────────────────────────────┤
│                            │                             │
│    히트맵                  │   브랜드 버블 차트            │
│    (Aspect x 감성)         │   (카테고리 내 포지셔닝)      │
│                            │                             │
├────────────────────────────┴─────────────────────────────┤
│                                                          │
│              경쟁사 SWOT 비교 테이블                       │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

### 5.3 워크시트 구성

| 워크시트 | 데이터소스 | 설명 |
|---------|-----------|------|
| 제품_선택기 | v_brand_dashboard_master | 브랜드/제품 드롭다운 필터 |
| 강점 (Strengths) | v_brand_dashboard_master | S: 긍정률 높은 Aspect 리스트 |
| 약점 (Weaknesses) | v_brand_dashboard_master | W: 부정률 높은 Aspect 리스트 |
| 기회 (Opportunities) | v_brand_dashboard_master | O: ML 연착륙 확률, CP Index 등 |
| 기회 (Opportunities) - 보조 | v_brand_dashboard_master | O 보조 지표 |

## 9. 대시보드 간 연결 구조

```
[시뮬레이터]  ──(입점 전 시뮬레이션)──→  등급 A/B/C/D 판정
                                              │
                                              ▼
[Safety Monitor]  ──(입점 후 모니터링)──→  경고/주의/정상 판정
                                              │
                                              ▼
[자가진단]  ──(개별 제품 분석)──→  SWOT + 경쟁 포지셔닝
```

시뮬레이터는 **입점 전** 제품의 연착륙 가능성을 사전 평가하고, Safety Monitor는 **입점 후** 기존 제품의 감성 위험을 지속 모니터링한다. 자가진단 대시보드는 개별 제품 단위의 심층 분석을 제공한다.

공통 데이터소스인 `v_brand_dashboard_master`가 Safety Monitor와 자가진단 대시보드를 연결하며, `v_category_scorecard_ref`가 시뮬레이터의 핵심 데이터를 담당한다.
