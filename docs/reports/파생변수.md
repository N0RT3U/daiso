## 1. Products DataFrame 파생변수 (6개)

### 1.1 review_count

- **설명**: 해당 제품에 달린 총 리뷰 개수
- **생성 방법**: `reviews_core.groupby('product_code').size()`
- **데이터 타입**: int
- **값의 범위**: 0 ~ 9,504
- **평균**: 340.84
- **활용 목적**: 제품 인기도 측정, engagement_score 계산에 사용

### 1.2 engagement_score

- **설명**: 제품의 전체적인 참여도를 나타내는 종합 지표
- **생성 방법**:

  ```
  engagement_score = 0.15 × likes + 0.30 × shares + 0.55 × review_count
  ```

  - 가중치는 각 지표의 변동계수(CV)를 기반으로 결정
  - review_count(CV 2.050) > shares(CV 1.884) > likes(CV 1.081) 순의 분별력
  - 변동계수가 높을수록 높은 가중치 부여
- **데이터 타입**: float
- **값의 범위**: 0 ~ 6,819.0 (평균 566.32)
- **활용 목적**: 제품의 종합적인 인기도 및 관심도 평가, 마케팅 우선순위 결정

### 1.3 cp_index

- **설명**: 가성비 지표 (Cost-Performance Index)
- **생성 방법**:

  ```
  cp_index = (engagement_score / price) × 1000
  ```

  - 1000을 곱하여 소수점 단위를 정수 범위로 정규화
- **데이터 타입**: float
- **값의 범위**: 0 ~ 6,776.30 (높을수록 가성비 우수)
- **활용 목적**: 가격 대비 인기도 평가, 갓성비 상품 발굴

### 1.4 first_review_date

- **설명**: 해당 제품의 첫 리뷰 작성일
- **생성 방법**: `reviews_core.groupby('product_code')['review_date'].min()`
- **데이터 타입**: date (YYYY-MM-DD)
- **값의 범위**: 리뷰가 없는 제품은 NULL (11건)
- **활용 목적**: 출시 시점 추정, review_density 계산 기반, 연착륙 상품 판별

### 1.5 review_density

- **설명**: 일평균 리뷰 발생 밀도 (지속적 인기도 지표)
- **생성 방법**:

  ```
  review_density = review_count / (last_review_date - first_review_date).days
                 = 0   (review_count = 0일 때, 기간 0일은 1일로 처리)
  ```
- **데이터 타입**: float
- **값의 범위**: 0 ~ 16.8363
- **활용 목적**: 시간 경과에 따른 꾸준한 관심도 평가, 연착륙 상품 발굴

### 1.6 risk_score

- **설명**: 품질 리스크 점수 (수동 관리 필드)
- **생성 방법**: 기존 데이터에서 보존, 신규 제품은 기본값 0.0
- **데이터 타입**: float
- **값의 범위**: 0.0 ~ (수동 입력)
- **활용 목적**: 품질 이슈 발생 제품의 리스크 수준 관리

## 2. Reviews DataFrame 파생변수 (6개)

### 2.1 review_length

- **설명**: 리뷰 텍스트의 문자 길이
- **생성 방법**: `reviews_df['text'].fillna('').str.len()`
- **데이터 타입**: int
- **값의 범위**: 0 ~ 996자 (평균 35.77자, 중간값 24자)
- **활용 목적**: 리뷰 품질 평가, 상세 리뷰 필터링

### 2.2 is_reorder

- **설명**: 재구매 리뷰 여부
- **생성 방법**:

  ```python
  is_reorder = reviews_df['text'].fillna('').str.startswith('재구매')
  ```
- **데이터 타입**: bool
- **통계**: 재구매 리뷰 95,536건 (29.57%), 일반 리뷰 227,578건 (70.43%)
- **활용 목적**: 고객 충성도 측정, 재구매율 분석 (RQ1 핵심 지표)

### 2.3 user_total_reviews

- **설명**: 해당 사용자가 작성한 총 리뷰 수
- **생성 방법**: `reviews_core.groupby('user_id').size()`
- **데이터 타입**: int
- **값의 범위**: 1 ~ 2,080
- **활용 목적**: 사용자 활동성 평가, user_activity_level 계산 기반

### 2.4 user_activity_level

- **설명**: 사용자 활동 수준 분류
- **생성 방법**:

  | 등급    | 조건          | 유저 수 (비율)     | 리뷰 수 (비율)       |
  | ------- | ------------- | ------------------ | -------------------- |
  | Newbie  | 1개 리뷰      | 5,554명 (21.8%)    | 5,554건 (1.7%)       |
  | Junior  | 2 ~ 5개 리뷰  | 8,935명 (35.0%)    | 28,188건 (8.7%)      |
  | Regular | 6 ~ 20개 리뷰 | 7,338명 (28.7%)    | 78,649건 (24.3%)     |
  | VIP     | ≥ 21개 리뷰  | 3,704명 (14.5%)    | 210,723건 (65.2%)    |

- **고유 사용자**: 25,531명
- **데이터 타입**: str (categorical)
- **활용 목적**: 사용자 세분화, 신뢰도 가중치 결정

### 2.5 reorder_user_avg_rating

- **설명**: 재구매 리뷰를 작성한 사용자의 평균 평점 (재구매 리뷰만 대상)
- **생성 방법**: `reorder_reviews.groupby('user_id')['rating'].mean()`
- **데이터 타입**: float
- **값의 범위**: 1.0 ~ 5.0 (평균 4.824) 또는 NaN
- **재구매 경험 사용자**: 14,035명 (55.0%)
- **활용 목적**: 재구매 고객의 만족도 평가

### 2.6 user_rating_tendency

- **설명**: 사용자의 평점 부여 성향 (재구매 리뷰 기준)
- **생성 방법**:

  | 성향            | 조건             | 비율               |
  | --------------- | ---------------- | ------------------ |
  | Always Positive | 평균 평점 ≥ 4.8 | 66.7% (215,630건)  |
  | Mostly Positive | 4.0 ~ 4.8        | 18.2% (58,782건)   |
  | No Reorder      | 재구매 경험 없음 | 11.8% (37,982건)   |
  | Balanced        | 3.0 ~ 4.0        | 3.1% (9,949건)     |
  | Critical        | < 3.0            | 0.2% (771건)       |

- **데이터 타입**: str (categorical)
- **활용 목적**: 사용자 성향 파악, 평점 신뢰도 평가

## 3. ERD 파이프라인 파생변수 (5개)

ERD 정규화 과정(`05_src/04_pipeline/transformer.py`, `05_src/04_pipeline/derived_features.py`)에서 리뷰 단위 데이터를 유저 단위로 집계하여 생성하는 변수이다.

### 3.1 review_tenure (Users_profile)

- **설명**: 유저의 리뷰 활동 기간 (첫 리뷰 ~ 마지막 리뷰 사이 일수)
- **생성 방법**:

  ```python
  user_dates = reviews_core.groupby('user_id')['review_date'].agg(['min', 'max'])
  review_tenure = (max - min).days
  ```

  - 리뷰가 1건인 유저는 0일
  - 최초 리뷰일과 최종 리뷰일의 날짜 차이
- **데이터 타입**: int
- **값의 범위**: 0 ~ 764일
- **활용 목적**: 유저 충성도 및 장기 활동 패턴 분석, VIP 유저 식별 보조 지표

### 3.2 user_avg_rating_reorder (Users_profile)

- **설명**: 유저별 재구매 리뷰 평균 평점 (Users_profile 테이블용, 재구매 없으면 0.0)
- **생성 방법**:

  ```python
  reorder_avg = reviews[reviews['is_reorder'] == True].groupby('user_id')['rating'].mean()
  user_avg_rating_reorder = reorder_avg.fillna(0.0).round(2)
  ```

  - Users_repurchase의 `reorder_user_avg_rating`과 동일 계산이지만, 재구매 없는 유저는 0.0으로 채움
- **데이터 타입**: float
- **값의 범위**: 0.0 ~ 5.0 (재구매 없는 유저는 0.0)
- **활용 목적**: 유저 프로필의 재구매 만족도 지표, user_rating_tendency 계산 기반

### 3.3 reorder_user_brand (Users_repurchase)

- **설명**: 유저별 브랜드 재구매 총 건수
- **생성 방법**:

  ```python
  # is_reorder=True인 리뷰만 대상
  reorder = reviews_core[reviews_core['is_reorder'] == True]
  # 동일 유저 × 동일 brand_id에 2건 이상 → 해당 리뷰들을 1로 표기 → 유저별 합산
  user_brand = reorder.groupby(['user_id', 'brand_id']).size().reset_index(name='cnt')
  reorder_user_brand = user_brand[user_brand['cnt'] >= 2].groupby('user_id')['cnt'].sum()
  ```

- **데이터 타입**: int
- **값의 범위**: 0 ~ 635
- **활용 목적**: 브랜드 충성 고객 식별, 유저별 브랜드 고착도 분석

### 3.4 reorder_user_category (Users_repurchase)

- **설명**: 유저별 카테고리 재구매 총 건수
- **생성 방법**:

  ```python
  # is_reorder=True인 리뷰만 대상
  reorder = reviews_core[reviews_core['is_reorder'] == True]
  # 동일 유저 × 동일 category_2(소분류)에 2건 이상 → 해당 리뷰들을 1로 표기 → 유저별 합산
  user_cat = reorder.groupby(['user_id', 'category_2']).size().reset_index(name='cnt')
  reorder_user_category = user_cat[user_cat['cnt'] >= 2].groupby('user_id')['cnt'].sum()
  ```

- **데이터 타입**: int
- **값의 범위**: 0 ~ 648
- **활용 목적**: 카테고리 충성 고객 식별, 유저별 카테고리 고착도 분석

### 3.5 reorder_user_avg_rating (Users_repurchase)

- **설명**: 유저별 재구매 리뷰 평균 평점
- **생성 방법**:

  ```python
  reorder = reviews_core[reviews_core['is_reorder'] == True]
  reorder_user_avg_rating = reorder.groupby('user_id')['rating'].mean().round(2)
  ```

- **데이터 타입**: float
- **값의 범위**: 1.0 ~ 5.0 (평균 4.824)
- **활용 목적**: 재구매 고객의 만족도 평가

### 3.6 변수 간 관계 요약

```
[reviews_core 기반 재계산]                        [ERD 테이블]

reviews_text.text.startswith('재구매')
    → is_reorder (리뷰)
        → user_avg_rating_reorder (유저)       ──→ Users_profile.user_avg_rating_reorder
        → user_rating_tendency (유저)          ──→ Users_profile.user_rating_tendency
        → reorder_user_avg_rating (유저)       ──→ Users_repurchase.reorder_user_avg_rating
        → 동일 brand_id에 2건+ 재구매
            → 유저별 합산                      ──→ Users_repurchase.reorder_user_brand
        → 동일 category_2에 2건+ 재구매
            → 유저별 합산                      ──→ Users_repurchase.reorder_user_category

reviews_core.groupby('user_id').size()
    → user_total_reviews (유저)               ──→ Users_profile.user_total_reviews
        → user_activity_level (유저)          ──→ Users_profile.user_activity_level

reviews_core.groupby('user_id')['review_date'].agg(max-min)
    → review_tenure (유저)                    ──→ Users_profile.review_tenure

※ Users_repurchase는 재구매 경험 유저(14,035명)만 포함. 비재구매 유저(11,496명)는 제외.
```

## 4. 주의사항

1. **결측치 처리**

   - Users_repurchase: 재구매 이력이 없는 유저는 테이블에서 제외된다. users_full에서는 reorder 3개 컬럼이 NaN이다
2. **계산 제약**

   - cp_index: price가 0인 경우 0으로 처리한다
   - review_density: review_count가 0이면 0, 리뷰 기간이 0일이면 1일로 처리한다
3. **해석 시 유의점**

   - engagement_score는 상대적 지표이므로 카테고리 간 직접 비교에 주의해야 한다
   - is_reorder는 텍스트 기반 판단이므로 실제 재구매와 차이가 발생할 수 있다
4. **user_id 식별 한계 (마스킹 기반 동일인 판단)**

   다이소몰은 리뷰 작성자의 닉네임을 마스킹하여 노출한다 (예: `min******`). 크롤링 시 이 `user_masked` 문자열을 키로 사용하여 `user_id`를 할당한다.

   - **마스킹 규칙**: 앞 3글자 공개 + 나머지를 `*`로 대체 (원래 ID 길이 보존)
   - **동일인 판단 기준**: `user_masked` 문자열이 완전히 같으면 같은 유저로 간주한다

   **구조적 한계:**

   | 위험 유형                  | 설명                                                                                                                |
   | -------------------------- | ------------------------------------------------------------------------------------------------------------------- |
   | 충돌 (다른 사람 → 합쳐짐) | 앞 3글자와 원래 ID 길이가 동일한 서로 다른 유저가 1명으로 합쳐질 수 있다. 데이터만으로는 검증이 불가하다                  |
   | 분리 (같은 사람 → 나뉨)   | 닉네임을 변경한 유저는 다른 사람으로 인식된다                                                                         |
   | `*` 유저 문제            | ID가 1글자인 모든 유저가 `*` 하나로 합쳐진다 (user_id=56, 765건 리뷰, 322개 제품 — 복수 인물이 합쳐진 것으로 추정) |

   **영향 범위**: user_id에 의존하는 모든 유저 파생변수(user_total_reviews, user_activity_level, user_avg_rating_reorder, reorder_user_avg_rating, user_rating_tendency, review_tenure, reorder_user_brand, reorder_user_category)에 영향을 준다
