# Part A: SLI_v2 방법론 및 검증

> **분석 기준:** SLI_v2 통합 후 최종 연착륙 **158개** / 비연착륙 **548개** (전체 706개)

---

## 1. 핵심 개념 가이드

분석에 사용된 주요 개념과 통계 용어를 쉽게 풀어서 설명한다.

### 1.1 SLI_v2 (Soft Landing Index v2)

기존 SLI(v1)가 3구간(early/mid/late) + 단순 비율 + polyfit 기울기를 사용한 반면, **SLI_v2**는 4구간 + 로그 차이 + 비대칭 기울기로 설계된 개선 버전이다.

* **polyfit :** 데이터 분석에서 데이터들의 흐름을 가장 잘 나타내는 직선이나 곡선을 찾아주는 수학적 도구. 복잡하게 흩어진 점들 사이에서 하나의 대표적인 법칙(수식)을 찾아낼 때 사용한다.

```
SLI v1: early(0~2) / mid(3~5) / late(6~12) → 비율 기반 (late/early >= 0.6)
SLI v2: early(0~2) / mid(3~5) / late1(6~8) / late2(9~12) → 로그 차이 기반
```

> **왜 4구간인가?**
> 6\~12개월을 하나로 묶으면, "6개월 차에는 유지되지만 10개월 이후 재상승"하는 **후기 가속** 패턴을 포착할 수 없다. late1(6\~8)과 late2(9~12)를 분리해서 **"후기에 다시 살아나는 제품"** 을 식별한다.

### 1.2 비대칭 기울기 (Directional Slope)

기존: `slope = polyfit` → 상승이든 하락이든 절대값으로 패널티
SLI_v2: **하락만 패널티**, 상승은 감점 안 함

```
slope = -5.0 (하락) → slope_neg = 5.0 (패널티 부과)
slope = +3.0 (상승) → slope_neg = 0.0 (감점 없음)
```

> **왜 비대칭인가?**
> 상승 추세는 "성장"이므로 연착륙에 부정적이지 않다. 기존 방식에서는 성장하는 제품도 기울기 절대값이 커서 불이익을 받았다. SLI_v2는 **하락만 위험 신호**로 보고, 상승은 자유롭게 허용한다.

### 1.3 후기 변동성 (Late Volatility CV)

후기 구간(6개월+)에서의 **변동계수(CV = 표준편차 / 평균)** 를 측정한다.

```
제품 A: 후기 리뷰 [50, 55, 48, 52] → CV = 0.05 (안정적)
제품 B: 후기 리뷰 [80, 10, 90, 5]  → CV = 0.95 (불안정)
```

> **왜 후기만 보나?**
> 초기(0~5개월)는 프로모션, 품절, 바이럴 등으로 자연스럽게 변동이 크다. 연착륙의 핵심은 **"후기에 안정적으로 유지되는가"** 이므로, 후기 데이터에서만 변동성을 측정한다.

### 1.4 RobustScaler + 가중합

6개 컴포넌트를 **RobustScaler**(중앙값 기준, 이상치에 강건)로 표준화한 후 가중합한다.

| 컴포넌트              | 가중치          | 의미               |
| --------------------- | --------------- | ------------------ |
| `mid_minus_early`   | **+0.20** | 초기→중기 유지    |
| `late1_minus_mid`   | **+0.25** | 중기→후기초 유지  |
| `late2_minus_late1` | **+0.35** | 후기 가속 (핵심)   |
| `log_total_reviews` | **+0.15** | 규모 기여도        |
| `slope_neg`         | **-0.05** | 하락 패널티        |
| `late_vol_cv`       | **-0.05** | 후기 변동성 패널티 |

> **왜 후기 가속(0.35)이 가장 높은가?**
> 프로젝트의 연착륙 정의가 "초기 붐 이후에도 무너지지 않고 **후기에 재상승하는 상품**"이기 때문이다. late2_minus_late1이 양수라면 "9~12개월 차에 다시 올라가는 제품"을 의미한다.

### 1.5 전문가 가중치란?

위 표의 가중치(0.20 / 0.25 / 0.35 / 0.15 / -0.05 / -0.05)는 **수학적 최적화가 아니라, 팀원(영호님)이 화장품 시장 경험과 데이터 패턴 관찰을 바탕으로 직접 설정한 값**이다. 이를 본 리포트에서 **"전문가 가중치"** 라고 부른다.

예를 들어 "후기에 다시 살아나는 게 연착륙의 핵심이니까 `late2_minus_late1`에 가장 높은 0.35를 주자"는 식의 **사람의 판단**으로 정해진 것이다.

> **"그럼 이 값이 정말 맞는 걸까?"** 라는 의문이 자연스럽게 생긴다. 이에 대한 답은 **[§20 부록](#20-부록-sli_v2-가중치-민감도-분석)** 에서 Grid Search 927개 조합을 컴퓨터로 전수 탐색하여, 전문가 가중치가 최적값과 단 2개(1.3%) 차이밖에 나지 않음을 데이터로 검증한다.

### 1.6 Over-Representation Ratio (OR)

**정의:**

- 특정 그룹(예: 립케어 카테고리)에서 연착륙 비율이 **전체 평균 대비 몇 배**인지를 나타내는 지표
- OR > 1: 과대표현 (연착륙 비율이 평균보다 높음)
- OR < 1: 과소표현 (연착륙 비율이 평균보다 낮음)

**계산 예시:**

```
전체 연착륙 비율 = 20.7%
립케어 연착륙 비율 = 44.4%

OR = (립케어 연착륙 비율 / 립케어 비연착륙 비율) / (전체 연착륙 비율 / 전체 비연착륙 비율)
   = (8/10) / (138/560) = 2.15

→ 립케어는 평균 대비 2.15배 연착륙 확률이 높음
```

> **왜 OR을 쓰나?**
> 단순히 "립케어 연착륙 8개"라고 하면 작아 보이지만, **모집단이 작은데도 연착륙 비율이 높다**는 것을 OR로 표현하면 "2.15배 유리"라는 인사이트로 전환된다.

### 1.7 Mann-Whitney U 검정

**정의:**

- 두 그룹(SL vs Non-SL)의 **중앙값(Median)** 이 통계적으로 유의하게 다른지 검정하는 비모수 검정 방법
- 정규분포를 가정하지 않아도 되므로, 리뷰 수나 참여도처럼 **편향된 분포**에 적합

> **왜 Mann-Whitney U를 쓰나?**
> t-test는 정규분포를 가정하는데, 우리 데이터는 **리뷰 수(극단값 존재)**, **가격(특정 구간 집중)** 같은 비정규 분포가 많다. Mann-Whitney U는 이런 경우에도 **중앙값 차이**를 안정적으로 검정한다.

### 1.8 Bonferroni 보정

**정의:**

- **다중 검정(Multiple Testing)** 을 수행할 때, 우연히 유의한 결과가 나올 확률(1종 오류)을 줄이기 위해 유의수준(α)을 조정하는 방법
- 즉, "여러 번 테스트할 때 생기는 **'운'** 의 요소를 더 엄격하게 **차단**하는 것"

**계산 예시:**

```
ABSA 8개 aspect를 검정할 때
일반 α = 0.05

Bonferroni 보정 α = 0.05 / 8 = 0.00625

→ p-value < 0.00625여야 유의하다고 판정
```

> **왜 Bonferroni 보정을 쓰나?**
> 8개 aspect를 검정하면, 우연히 1~2개는 p < 0.05가 나올 수 있다. 보정을 통해 **진짜 유의한 차이**만 걸러낸다.

### 1.9 Cohen's d

**정의:**

- 두 그룹 간 평균 차이를 **표준편차 단위**로 표현한 효과 크기(Effect Size) 지표
- "그래서 그 차이가 얼마나 큰데?"를 나타내는 단위

**해석 기준:**

```
|d| < 0.2  → 작은 효과
|d| < 0.5  → 중간 효과
|d| < 0.8  → 큰 효과
|d| >= 0.8  → 매우 큰 효과
```

> **왜 Cohen's d를 쓰나?**
> p-value는 "차이가 있다/없다"만 알려주지만, Cohen's d는 **"그 차이가 실무적으로 의미 있는 수준인가?"** 를 판단하게 해준다.

### 1.10 Lift (성분 분석)

**정의:**

- 특정 성분이 SL 그룹에서 **얼마나 과대표현되었는지** 나타내는 지표
- "특정 성분이 효자 상품에 얼마나 몰려 있는가?"를 보는 지표

**계산 예시:**

```
성분 A의 SL 출현율 = 3.4%
성분 A의 Non-SL 출현율 = 0.0%

Lift = 3.4% / 0.0% → ∞ (완전히 SL만 보유)
```

> **왜 Lift를 쓰나?**
> "당호박추출물"처럼 SL에만 나타나는 성분을 발견하면, **"이 성분이 연착륙에 기여하는가?"** 라는 가설을 세울 수 있다.

![DTW 클러스터링 패턴](../../02_outputs/01_figures/SLI/01_dtw_clustering.png)

> **해석:**
>
> * **이 그래프가 보여주는 것:** 706개 상품의 월별 리뷰 패턴을 AI(DTW 알고리즘)가 자동으로 **4가지 유형**으로 분류한 결과이다. 각 서브플롯이 하나의 유형이며, 선 하나가 제품 하나의 리뷰 추이이다.

>   - **급락형:** 출시 직후 반짝 인기 후 급격히 떨어지는 패턴 → 비연착륙
>   - **초기집중형:** 첫 1~2개월에만 리뷰가 쏠리고 이후 거의 없는 패턴 → 비연착륙
> * **핵심:** 후기상승형 + 안정유지형이 연착륙 후보이며, 이 그래프에서 선들이 뒤쪽(오른쪽)에서 올라가거나 유지되는 유형이 "좋은 제품"이다.

### 방법론 (2) Kaplan-Meier 생존곡선

![Kaplan-Meier 생존곡선](../../02_outputs/01_figures/SLI/02_survival_curves.png)

> **해석:**
>
> * **이 그래프가 보여주는 것:** 제품이 시간이 지나면서 얼마나 "살아남는지"를 보여주는 **생존 곡선**이다. 의학에서 환자의 생존율을 추적하는 방법을 제품에 적용한 것이다.
> * **X축:** 피크(가장 리뷰가 많았던 달) 이후 경과 월수
> * **Y축:** 생존 확률 (= 아직 리뷰가 활발하게 달리고 있는 제품의 비율). 1.0이면 전부 살아있고, 0.0이면 전부 사망.
> * **"사망" 기준:** 가장 리뷰가 많았던 달 대비 **30% 이하**로 **2개월 연속** 떨어지면 "이 제품은 시장에서 사라졌다"고 판정한다.
> * **곡선 읽는 법:** 곡선이 **천천히 내려갈수록** 오래 버티는 제품이 많다는 뜻이다. 곡선이 급격히 뚝 떨어지는 구간은 "그 시점에 많은 제품이 한꺼번에 사망했다"는 의미이다.

## 5. Step 1: SLI_v2 규칙기반 교체

### 무엇을 변경했는가?

기존 `rule_based_soft_landing()` 함수를 4개의 SLI_v2 컴포넌트 함수로 교체했다.

**기존 (3구간 단순 비율):**

```python
early = counts[:3].mean()
mid = counts[3:6].mean()
late = counts[6:].mean()
retention_ok = (late/early >= 0.6) and (mid/early >= 0.7)
slope = np.polyfit(range(len(counts)), counts, 1)[0]
slope_ok = slope > -0.5
```

**변경 후 (SLI_v2 — 4구간 + 비대칭 기울기):**

```python
# 4구간 중앙값(median) + log1p 차이
mid_minus_early   = log1p(mid_median)   - log1p(early_median)
late1_minus_mid   = log1p(late1_median) - log1p(mid_median)
late2_minus_late1 = log1p(late2_median) - log1p(late1_median)  # 후기 가속

# 비대칭 기울기 (하락만 패널티)
slope_neg = -slope if slope < 0 else 0.0

# 후기 변동성 (6개월+ CV)
late_vol_cv = std(late_counts) / mean(late_counts)

# RobustScaler 표준화 + 가중합 → SLI_v2 > 0이면 연착륙
```

### SLI_v2 규칙기반 결과

| 항목               | 기존 (3구간)  | **SLI_v2 (4구간)** | 변화           |
| ------------------ | ------------- | ------------------------ | -------------- |
| 규칙기반 연착륙 수 | 218개 (30.9%) | **284개 (40.2%)**  | +66개 (+30.3%) |
| SLI_v2 중앙값      | —            | 0.0315                   | —             |
| SLI_v2 > 0 비율    | —            | 40.2%                    | —             |

> **해석:** SLI_v2는 기존보다 66개 더 많은 상품을 규칙기반 연착륙으로 판정했다. 이는 SLI_v2가 **후기 가속(late2_minus_late1)** 패턴을 추가로 포착하기 때문이다. 기존 3구간 규칙에서는 "late 전체 평균"이 낮아서 탈락했지만, 4구간으로 나누면 "9~12개월에 재상승"하는 제품이 추가로 포착된다.

![SLI_v2 유지력 분포](../../02_outputs/01_figures/SLI/08_03_retention_distribution.png)

> **해석:**
>
> * **X축 (잔존율):** 제품의 출시 후 최대 리뷰 수(피크) 대비 최근 3개월간의 평균 리뷰 수 비율이다. 1.0에 가까울수록 피크 때의 인기를 현재까지 잘 유지하고 있음을 의미한다.
> * SL 상품의 중앙값(0.371)은 Non-SL 상품(0.175)보다 약 2배 이상 높다. 이는 연착륙 상품이 초기 붐이 지나간 후에도 피크 대비 약 37% 이상의 리뷰를 꾸준히 생성하며 시장에 성공적으로 안착했음을 입증한다. 그래프 중앙의 **연두색 음영 구역(30~50%)** 은 SL 상품들이 가장 밀집된 구간이다. 우리 프로젝트의 SLI 로직에서 연착륙을 판별하는 핵심 기준선이 이 부근에서 형성된다. 반면 Non-SL 상품은 대부분 20% 미만(빨간색 점선 왼쪽)에 몰려 있어 극명한 대조를 이룬다.

## 6. Step 2: ML 피처 추가

### 추가된 SLI_v2 도메인 피처 (6개)

기존 시계열 피처(14개)에 SLI_v2 컴포넌트 6개를 추가하여 총 **21개** 시계열 피처로 확장했다.

| 피처                  | 의미                         | 기존 피처와의 차이               |
| --------------------- | ---------------------------- | -------------------------------- |
| `sli_v2_score`      | SLI_v2 최종 점수             | 6개 컴포넌트의 가중합            |
| `mid_minus_early`   | 초기→중기 유지 (log1p 차이) | 기존 `ts_retention_mid`는 비율 |
| `late1_minus_mid`   | 중기→후기초 유지            | 기존에는 없음 (6~8개월 분리)     |
| `late2_minus_late1` | **후기 가속**          | 기존에는 없음 (9~12개월 분리)    |
| `slope_neg`         | 하락 패널티                  | 기존 `ts_slope`는 양방향       |
| `late_vol_cv`       | 후기 변동성                  | 기존 `ts_cv`는 전체 기간       |

### ML 모델 성능

| 지표           | 기존   | **SLI_v2 통합 후** | 변화   |
| -------------- | ------ | ------------------------ | ------ |
| OOF F1         | 0.8957 | **0.8705**         | -0.025 |
| OOF AUC        | 0.9851 | **0.9827**         | -0.002 |
| Precision (SL) | 0.84   | **0.82**           | -0.02  |
| Recall (SL)    | 0.96   | **0.92**           | -0.04  |
| Train-Val Gap  | 0.069  | **0.132**          | +0.063 |

> **해석:** ML 성능이 소폭 하락했다 (F1: 0.90 → 0.87). 이는 Pseudo Label이 152→171개로 증가하면서 학습 데이터의 양성 경계가 넓어진 영향이다. AUC 0.98은 여전히 우수한 수준이며, Gap 0.13도 허용 범위이다. **성능 하락보다 피처 기여도가 더 중요한 포인트**이다.

![LightGBM 혼동행렬](../../02_outputs/01_figures/SLI/04_ml_confusion_matrix.png)

> **해석:**
>
> * - **좌상 (True Positive):** 모델이 "연착륙"이라 예측했고, 실제로도 연착륙 → 정답 (맞춤)
>   - **우상 (False Positive):** 모델이 "연착륙"이라 예측했지만, 실제로는 비연착륙 → 오탐 (빗나감)
>   - **좌하 (False Negative):** 모델이 "비연착륙"이라 예측했지만, 실제로는 연착륙 → 놓침 (가장 아쉬운 실수)
>   - **우하 (True Negative):** 모델이 "비연착륙"이라 예측했고, 실제로도 비연착륙 → 정답 (맞춤)
> * **핵심 수치:**
>   - **Recall 92%:** 진짜 연착륙 제품 100개가 있으면 그 중 92개를 잡아낸다. 8개만 놓친다.
>   - **Precision 82%:** 모델이 "이건 연착륙이야!"라고 예측한 것 중 82%가 실제로 연착륙이다. 18%는 오탐(잘못된 예측)이다.
> * **비즈니스 의미:** Recall이 높다는 것은 "효자 상품을 거의 놓치지 않는다"는 뜻이므로, 상품 기획 의사결정에 신뢰할 수 있는 수준이다.

## 9. RQ 결론

### 핵심 인사이트

1. **SLI_v2 규칙기반은 기존 대비 더 관대하지만 합리적** — 284개(+30.3%)로 후기 가속 패턴을 추가 포착. 다만 최종 만장일치에서는 158개(+8.2%)로 수렴하여, 4개 방법론 합의 구조가 **자동 필터링** 역할을 수행

   - SLI_v2 중앙값 0.0315로, threshold=0 기준이 적절
2. **SLI_v2 컴포넌트가 ML에 유의미하게 기여** — `slope_neg`(6위, 0.1474)과 `late_vol_cv`(7위, 0.1457)가 ABSA 피처보다 높은 SHAP 중요도

   - "하락 방향성"과 "후기 안정성"이 연착륙 판별의 핵심 보조 신호
3. **스킨케어 후기 재구매 패턴이 SLI_v2에서 더 잘 포착** — 스킨케어 비율 65.8% → 68.4%, 본셉/바세린 신규 진입

   - 일상 소모품의 9~12개월 재구매 사이클이 late2_minus_late1에 반영
4. **ML 성능은 소폭 하락하나 허용 범위** — F1 0.90 → 0.87, AUC 0.99 → 0.98

   - Pseudo Label 확대(152→171)에 의한 자연스러운 변동

### 14.3 시계열 검색트렌드 비교분석 (앵커 기반 v4)

#### 왜 앵커가 필요한가? (방법론 설명)

네이버 DataLab API는 한 번에 최대 5개 검색어를 비교할 수 있고, 결과는 **그 5개 사이에서의 상대적 비율(ratio)** 로 나온다. 문제는 배치가 다르면 비교 기준이 달라진다는 것이다.

> **비유:** 학교마다 시험 난이도가 다르면, A학교 80점과 B학교 80점을 단순 비교할 수 없는 것과 같다.
> **해결:** 모든 학교에 같은 "기준 시험(앵커)"을 치르게 한 뒤, 그 기준 대비 점수로 환산하면 학교 간 비교가 가능하다.

**앵커 제품:** 다이소 딥 클렌징 폼 (product_code=1035082)

- 25개월 전 시점 데이터 존재, 0값 없음
- 변동계수(CV)=0.253으로 안정적, 다이소 대표 제품

**배치 구성:** 앵커 1개 + 다른 제품 4개 = 5개 그룹/배치 → 총 177배치 호출
**정규화:** 각 배치에서 `제품 ratio / 앵커 ratio` → 앵커=1.0 기준의 통일된 스케일

#### 데이터 개요

| 구분             | 제품 수 | 시계열 행 수 | 분석 대상 (>=10시점) |
| ---------------- | ------- | ------------ | -------------------- |
| **SL**     | 150개   | 3,222행      | 138개                |
| **Non-SL** | 513개   | 8,698행      | 440개                |

- **수집 기간:** 2024-01 ~ 2026-01 (25개월, 월 단위)
- **데이터 소스:** 네이버 DataLab 검색어 트렌드 API + 앵커 정규화

#### 분석 두 가지 관점

| 방법                                  | 무엇을 비교?                           | 계산                             | 알 수 있는 것                      |
| ------------------------------------- | -------------------------------------- | -------------------------------- | ---------------------------------- |
| **방법 A: 앵커 대비 절대 비교** | "SL 제품이 실제로 더 많이 검색되는가?" | 앵커 대비 ratio 그대로 비교      | 검색량의**절대적 크기** 차이 |
| **방법 C: 형태(Shape) 비교**    | "SL 제품의 검색 추이가 더 안정적인가?" | 제품별 최대값=1로 정규화 후 비교 | 시계열의**모양/패턴** 차이   |

#### 종합 비교 (방법 C: 형태 비교)

> **용어 설명:**
>
> - **변화율:** 수집 기간 후반의 검색지수가 전반 대비 얼마나 변했는지. -0.031은 "거의 변화 없음", -0.416은 "42% 하락"
> - **잔존율:** 가장 검색이 많았던 시점 대비 최근 3개월이 몇 % 수준인지. 0.371은 "피크의 37% 유지"
> - **CV(변동계수):** 검색량이 얼마나 들쑥날쑥한지. 낮을수록 안정적 (0.5 이하면 매우 안정, 1.0 이상이면 불안정)

| 지표                    | SL (138개)      | Non-SL (440개) | 차이                       | p-value    |
| ----------------------- | --------------- | -------------- | -------------------------- | ---------- |
| 트렌드: 상승            | 29.0%           | 32.3%          | -                          | -          |
| **트렌드: 안정**  | **34.1%** | 10.5%          | **SL +23.6%p**       | -          |
| 트렌드: 하락            | 37.0%           | 57.3%          | SL -20.3%p                 | -          |
| **중앙값 변화율** | **-3.1%** | -41.6%         | **SL이 38.5%p 완만** | 0.003**    |
| **중앙값 잔존율** | **37.1%** | 17.5%          | **SL 2.1배**         | 0.001**    |
| **중앙값 CV**     | **0.661** | 0.970          | **SL 1.5배 안정적**  | < 0.001*** |

> **핵심 (쉽게 말하면):**
>
> - SL 제품은 검색량이 **거의 줄지 않고** (-3%), Non-SL은 **절반 가까이 줄어들었다** (-42%)
> - SL 제품은 가장 인기 많았을 때의 **37%가 유지**되지만, Non-SL은 **17%만 남는다**
> - SL 제품의 검색량 변동이 **1.5배 더 안정적**이다

![3대 핵심 지표 대시보드](../../02_outputs/01_figures/SLI/08_07_three_metrics_dashboard.png)

**차트 읽는 법:**

- (a) 변화율 박스플롯: 박스의 가운데 선이 중앙값. SL(파랑)은 0 근처 = 거의 변화 없음. Non-SL(빨강)은 아래쪽 = 크게 하락
- (b) 잔존율 박스플롯: SL이 더 높은 위치 = 피크 대비 더 많이 남아있음
- (c) CV 박스플롯: SL이 더 낮은 위치 = 변동이 적고 안정적

### 14.4 종합 인사이트 (9개)

#### 1. 연착륙 제품은 검색량이 유의하게 높다 (p < 0.001)

- 제품당 평균 2.4배, 중앙값 2.1배 차이
- 특히 **쇼핑 채널에서 8.2배 차이** → 구매 전환 검색이 활발
- **액션:** 검색 가시성이 높은 제품을 연착륙 후보로 조기 식별

#### 2. "얼마나 많이"보다 "얼마나 꾸준히"가 핵심 (v4 신규)

- 앵커 기반 시계열 분석 결과, **절대적 검색 크기는 SL ≈ Non-SL** (유의차 없음)
- 하지만 **잔존율 2.1배, 변화율 38.5%p 차이, CV 1.5배** — 형태 지표에서 압도적 차이
- **연착륙 = "많이 검색되는 것"이 아니라 "꾸준히 검색되는 것"**

#### 3. 1K~2K 가격대가 "연착륙 스윗 스팟"

- 이 가격대에서 SL/Non-SL 검색량 격차 **6.7배**로 최대
- **액션:** 1K~2K 데일리 스킨케어 제품 중점 입점

#### 4. SL 제품의 34%가 "안정" 검색 패턴 (v4 신규)

- SL 안정 34.1% vs Non-SL 10.5% (3.2배)
- SL은 검색량 감소가 -3%에 불과하지만, Non-SL은 -42% 급락
- **액션:** 출시 6개월 후 검색 트렌드가 "안정"인 제품은 연착륙 후보로 조기 확정

#### 5. 카테고리별 연착륙 확률이 다르다

- **스킨케어(클렌징+팩+기초):** SL 비중 56.2% → 연착륙에 유리
- **립메이크업/치크:** Non-SL 편향 → 연착륙에 불리
- **아이메이크업:** SL 잔존율이 Non-SL의 4.0배 — SL이면 극강의 유지력

#### 6. 검색량 ≠ 연착륙 (Non-SL 고검색 제품 존재)

- 더마블록 PDRN 라인, 더페이스샵 스밈 라인 = Non-SL이지만 10만+ 검색량
- **검색량의 "절대값"보다 "지속성"이 핵심**

#### 7. 동일 브랜드 내에서도 SL/Non-SL 격차 존재

- 에이솔루션, 어퓨 = 같은 브랜드인데 SL 제품이 7~8배 높은 검색량
- **브랜드 파워가 아닌 "개별 제품의 바이럴 성공"** 이 연착륙 결정
- **액션:** 브랜드 단위가 아닌 개별 SKU 단위 검색 모니터링

#### 8. 블로그는 공통, 쇼핑이 차별화 포인트

- 두 그룹 모두 블로그 90%+ (공통)
- SL 쇼핑 비중 7.3% vs Non-SL 2.1% (3.5배)
- **액션:** SL 후보 제품은 쇼핑 검색 최적화(가격 비교 노출) 강화

#### 9. 키워드 전략 재확인

- "후기" > "리뷰" (1.5배) — 두 그룹 공통
- 브랜드명 강하면 "다이소" 키워드 불필요
- Non-SL 제품이 "후기" 키워드 의존도 더 높음 (1.56x vs 1.09x)
- **액션:** SL 제품은 브랜드 SEO, Non-SL 제품은 "후기" 키워드 SEO

## 15. 축6: 성분 심층 분석

연착륙 제품과 비연착륙 제품 간의 **성분 계열(Type)** 및 **효능(Effect)** 차이를 분석했다.

### 15.1 성분 계열(Type) 분포 비교

![성분 계열 분포](../../02_outputs/01_figures/SLI/10_04_ingredient_type_comparison.png)

> **해석:**
>
> * **연착륙 특화 계열 (양수 방향):**
>   - **Chelator(+16.1%p):** 킬레이트제 — 물 속 금속 이온을 잡아 제품 안정성을 높이는 성분. 스킨케어 기초 처방에 필수적이다.
>   - **Water(+14.3%p):** 수분 베이스 — 수분 함량이 높은 제품(로션, 토너 등)에 많이 포함된다.
>   - 즉, 연착륙 제품은 **"물 베이스의 순한 기초 스킨케어"** 성분 구성을 가지고 있다.
> * **비연착륙 특화 계열 (음수 방향):**
>   - **Colorant(-22.2%p):** 색소 성분 — 립스틱, 아이섀도 등 색조 화장품의 핵심 성분이다.
>   - **Acid(-16.9%p):** 산(Acid) 계열 — 필링, 각질 제거 등에 사용되는 강한 성분이다.
>   - 즉, 색조 화장품과 기능성 강한 제품은 연착륙에 불리하다.
> * **비즈니스 시사점:** 다이소 입점 시 순한 수분 베이스 스킨케어를 우선 선별하면 연착륙 확률이 높아진다.

### 15.2 성분 효능(Effect) 분포 비교

![성분 효능 분포](../../02_outputs/01_figures/SLI/10_05_ingredient_effect_comparison.png)

> **해석:**
>
> * **연착륙 특화 효능 (양수 방향):**
>   - **Preservative(+6.4%p):** 방부제 — 수분 함량이 높은 스킨케어 제품은 세균 증식을 막기 위해 반드시 방부 성분이 필요하다. 방부제 비율이 높다는 것 자체가 "이 제품은 수분 베이스 스킨케어"라는 간접 증거이다.
> * **비연착륙 특화 효능 (음수 방향):**
>   - **Color(-22.2%p):** 발색 — 색을 내는 역할의 성분. 색조 화장품(립, 아이섀도)에 집중된다.
>   - **Texture(-7.3%p):** 텍스처 — 제품의 질감을 만드는 성분. 파운데이션, 프라이머 등에 많다.
> * **Type 분석과의 연결:** Type에서 "Colorant가 비연착륙에 많다"고 나왔는데, Effect에서도 "Color 효능이 비연착륙에 많다"고 나와 **교차 검증**이 된다.

### 15.3 Type x Effect 히트맵

![Type x Effect 히트맵](../../02_outputs/01_figures/SLI/10_06_type_effect_heatmap.png)

> **해석:**
>
> * **X축:** 성분 효능 (Effect) — Preservative, Base, Color, Texture 등

## 20. 부록: SLI_v2 가중치 민감도 분석

### 20.1 1차원 민감도 분석

6개 가중치를 각각 독립적으로 ±50% 범위에서 변화시키며, 나머지는 전문가 가중치로 고정한 상태에서 만장일치(4/4) 수와 3표 이상 수의 변화를 측정했다.

![1차원 민감도 분석](../../02_outputs/01_figures/SLI/09_01_sensitivity_1d.png)

> **해석:**
>
> * **X축:** 가중치 값 (원래 값을 중심으로 ±50% 변화)
> * **Y축:** 만장일치(4/4) 수 — 4개 방법론 모두 연착륙이라 동의한 제품 수
> * **읽는 법:** 선이 **수평에 가까울수록** 해당 가중치를 바꿔도 결과가 거의 변하지 않는다는 뜻이다(= 강건함).
> * **핵심 발견:**
>   - 6개 가중치 모두에서 만장일치 수가 **±5개 이내**로 안정적이다.
>   - **`late2_minus_late1`(후기 가속)** 과 **`late1_minus_mid`(중기→후기 유지)** 에서 변동이 가장 크다 → 이 두 가중치가 SLI_v2의 핵심 파라미터이다.
>   - **`slope_neg`과 `late_vol_cv`(패널티 가중치)** 는 변화에 둔감하다 → 강건하다.
> * **의미:** 전문가 가중치를 약간 다르게 설정했더라도 최종 결과는 크게 달라지지 않았을 것이다.

> * **의미:** "사람이 정한 값"과 "컴퓨터가 최적화한 값"이 거의 일치한다. 전문가의 도메인 지식이 데이터로도 뒷받침된다는 강력한 증거이다.

### 20.3 2차원 상호작용 분석

가장 민감한 두 가중치(`late2_minus_late1` x `late1_minus_mid`)의 상호작용을 히트맵으로 시각화했다.

![2D 상호작용 히트맵](../../02_outputs/01_figures/SLI/09_03_interaction_heatmap.png)

> **해석:**
>
> * **X축:** `late2_minus_late1`(후기 가속) 가중치 값
> * **Y축:** `late1_minus_mid`(중기→후기 유지) 가중치 값
> * **색상:** 진한 색 = 만장일치 수가 많음(좋음), 연한 색 = 만장일치 수가 적음(나쁨)
> * **핵심 발견:**
>   - 전문가 가중치가 **진한 색 영역(최적 구간) 근처**에 위치한다.
>   - 두 가중치의 합이 **0.55~0.65 범위**에서 가장 좋은 성능을 보인다.

## 21. 한계점 및 향후 과제

### 데이터의 한계

1. **12개월 제한**: MAX_MONTH=12로 설정해서, 12개월 초과의 장기 연착륙 패턴은 미관측

### 검색트렌드 한계

2. **키워드 한계**

   - 제품명 + 브랜드명 조합만 수집
   - 카테고리 키워드("선크림 추천") 미수집
3. **인과관계 한계**

   - 검색량이 높아서 연착륙인지, 연착륙이라 검색량이 높은지 방향성 불분명
   - 교란변수(브랜드 파워, 출시 시점, 마케팅 투자) 미통제
4. **카테고리 내 표본 크기**

   - 개별 카테고리 내 비교 시 표본 부족으로 통계적 유의성 한계
   - 특히 치크/하이라이터 SL 4개, 립케어 SL 8개 등
5. **앵커 의존성**

   - 모든 비교가 단일 앵커(딥 클렌징 폼) 기준이므로, 앵커 선택에 따라 결과가 달라질 수 있음
   - 향후 복수 앵커 교차 검증으로 보완 가능
